Get-ZabbixAuthCode

Authorize use of the API.

Login and receive an authorization token.
User must have a Zabbix logon.
If security settings for the user have changed, a new authorization token will be required.
If username and password are omitted they will be prompted for.

Syntax 

Get-ZabbixAuthCode [[-Username] <String>] [[-Password] <SecureString>] [-storeToProfile] [<CommonParameters>]

Parameters

Name Alias Type Description Required? Pipeline Input Default Value
-Username String Username to log in with. false false
-Password SecureString Password tl log in with. This MUST be a secure string. false false
-storeToProfile SwitchParameter Stores the authorization token in a configuration file in the directory .zabbix in the user's profile. false false False

Outputs

The output type is the type of the objects that the cmdlet emits.

  • The authorization token as a string.

Invoke-ZabbixAPI

Make a custom call to the zabbix API.

This method allows you to make a customized call to the API.
There are many parameters for the API calls. Not all of them are in the module functons. This is for simplicity sake.

Syntax 

Invoke-ZabbixAPI [-Method] <String> [-params] <PSObject> [[-authcode] <String>] [<CommonParameters>]

Parameters

Name Alias Type Description Required? Pipeline Input Default Value
-Method String This is the method to call. i.e. 'host.get' true false
-params PSObject This is an object containing the parameters to use. true false
-authcode String false false

Outputs

The output type is the type of the objects that the cmdlet emits.

  • Return the result of the call. This is the complete response. It is up to you to display or use the data.

Examples

EXAMPLE 1 
This example calls the hostgroup.get method and returns groups with a hosts property containing hostid and name.
$params = @{
    selectHosts = @(
        "hostid",
        "name"
    )
}
$response  = Invoke-ZabbixAPI -Method "hostgroup.get" -params $params
EXAMPLE 2 
This example shows how to search hosts by host tags. Only output hostid and host name.
$params = @{
    output = @(
        "hostid",
        "name"
    )
    selectTags = "extend"
    tags = @(
        @{
            tag = "Host name"
            value = "Linux server"
            "operator" = 1
        }
    )
}
$response = Invoke-ZabbixAPI -Method "host.get" -params $params
